Card Render Comparison — Iteration 1

Our renders vs real MTG cards — spot the differences, fix, repeat

Where we are: Phase 2C (Card Renderer) of the MTGAI project — building an automated pipeline that renders custom Magic: The Gathering cards as print-ready images. We have 66 cards with AI-generated art, card data (names, costs, rules text, etc.), and real M15 frame templates from Card Conjurer.

What this is: First render pass. We composite art + frame + text into final card images. This page compares our renders against real MTG cards from Scryfall to identify visual differences. Each iteration fixes the biggest issues, re-renders, and compares again.

Goal: Near-pixel-perfect match to real MTG card layout — correct font sizes, positions, spacing, symbol rendering, and overall feel.

Known Issues — Iteration 1

CRITICAL Mana cost symbols missing
No mana symbols rendered in the top-right of the name bar. The {1}{W} cost should appear as colored circles.
CRITICAL Rules text too small / wrong position
Oracle text is barely readable. Font size appears much too small, and text may not be filling the text box properly. Compare to real cards where rules text is prominent and fills the space.
CRITICAL Set symbol missing
No set symbol visible on the right side of the type bar. Should show the ASD vortex symbol colored by rarity.
MAJOR P/T box text too small
Power/toughness is present but barely visible. Should be large, bold text centered in the P/T box.
MAJOR Collector info missing or invisible
The bottom collector bar (card number, set code, artist credit) is not visible.
MINOR Card name font size
May need adjustment — compare to real cards for proper weight and size.
MINOR Flavor text separator
Check if the italic flavor text separator line renders correctly between oracle and flavor text.
