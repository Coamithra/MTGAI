## Tunable knobs (default → allowed range)

{knob_listing}

Rarity weights are expressed per ~277-card premier set; they are scaled to this
set's size automatically. The multicolor / colorless / creature knobs are
fractions of that rarity's cards. The non-creature bias knobs are relative weights
(instant / sorcery / enchantment / artifact) that split each color's non-creature
slots; raise artifact or enchantment to skew the type mix.

The card-subtype knobs refine the coarse types into the textures real sets carry.
The recurring ones (equipment, vehicles, auras, artifact creatures) are counts per
~277-card set, scaled automatically — raise the one your theme leans on (a gear
theme wants more equipment; a sci-fi theme more artifact creatures; an enchantments
theme more auras). `irregular_subtype_count` is how many deciduous "specials"
(Sagas, Classes, Shrines, enchantment creatures) to weave in — most sets carry 0-2,
so keep it small unless the theme is built on one. Use `irregular_subtypes` to name
WHICH ones fit the setting, most-fitting first (a myth/history theme → saga; a
devotion/temple theme → shrine; an eerie/enchantment theme → enchantment_creature;
a colored-artifact theme → colored_artifact), and set `irregular_subtype_count` to
match how many you listed. Leave `irregular_subtypes` empty when the theme has no
real preference — a seeded RNG then picks for you. `subtype_jitter` randomizes the
counts a little; leave `subtype_seed` alone.

## Derived slot budgets (this set's size)

These are the approximate per-rarity slot counts your skeleton will have at this
set size, and how each rarity splits across mono / multicolor / colorless slots:

{budget_listing}

Propose cycles and signposts that FIT these budgets. A multicolor cycle consumes
slots from its rarity's multicolor budget: a `pairs10` cycle needs 10 multicolor
slots of its rarity, an `allied5` / `enemy5` needs 5. `signposts_per_pair` is
bounded by the UNCOMMON multicolor budget above (N per pair x 10 pairs must fit
it) — on a small set that budget is only a handful, so 2 per pair will only cover
the first few pairs. A cycle that doesn't fit its budget is dropped, so size your
proposals to the numbers above rather than relying on reconciliation.

## Cycle spans

A cycle's `span` chooses how its members spread across colors:
{cycle_spans}

Each cycle needs: a short `id` (lowercase, no spaces), a `name`, a `rarity`, a
`span`, a `card_type` (creature / instant / sorcery / enchantment / artifact /
land), an optional `cmc_target` (0 for lands), and a one-line `template` — the
shared design brief every member follows so the family reads as a set.
