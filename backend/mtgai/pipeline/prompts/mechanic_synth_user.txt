## Your task

1. **Apply a consensus filter.** Act on issues that AT LEAST 2 members raised (or that a single member raised but is plainly, objectively correct — e.g. an example card that does not use the mechanic at all). DISCARD issues only one member raised that are matters of taste — those are likely false positives. Lone strong opinions are noise; agreement is signal.

2. **Revise the mechanic in place** to fix the consensus issues, keeping everything that already works. This is your primary tool — fixing is better than rejecting. Improve wording, sharpen the decision the mechanic creates, cut bloat, rewrite a broken example card. Respect the OFF-LIMITS list (never rename, never touch the rarity distribution or keyword_type, never optimise for theme). The `revised_mechanic` must be COMPLETE and well-formed against the same schema — every field present, exactly two example cards that use the mechanic at contrasting rarities.

   Fix the issue **in the text itself, not just in your notes** — a changelog claiming a fix you did not actually make is worse than no fix:
   - The `reminder_text` is the keyword's real definition. It must be self-contained and rules-complete: it states exactly what happens using only defined Magic terms and actions. NO undefined verbs or nouns (e.g. "switch its mode", "gain an Energon"), NO bracket placeholders (e.g. "[Cost]"). If the mechanic relies on a counter or token, the reminder text itself must say how that counter/token is created, what it is, and what it does — don't push the definition into the example cards and leave the reminder vague.
   - The `reminder_text` and BOTH example cards must be in **lockstep**: each example uses the keyword by name and behaves EXACTLY as the reminder text defines it. If you change the definition, change the examples to match, and vice-versa. If a reviewer flagged the wording, the revised wording must be genuinely different — don't re-emit the same flawed phrase.
   - **Never introduce a placeholder.** No `[cost]`, `[target]`, `[keyword]`, etc. Write the concrete cost/target, or put the cost on the keyword (`Crew 2`, `Equip {2}`).
   - **Never delete a definition the draft already had.** If the draft's reminder defined a token or counter (e.g. "a Scrap token is a colorless artifact with '{T}, Sacrifice this: Add {C}'"), KEEP that definition — stripping it leaves an undefined referent. Improve it; don't remove it.
   - **Plain text only** — no markdown, no asterisks, no emphasis formatting, in any field.
   - **Use correct Magic templating:**
     - Keyword ability with a cost → put the cost on the keyword (`Crew 2`, `Equip {2}`); the reminder defines the full ability. Not `{T}, pay [cost]`.
     - Activated ability → `[cost]: [effect]` (the cost may include `{T}`).
     - Optional ETB with a cost → `When this creature enters, you may [cost]. If you do, [effect].` — not `When this enters, [cost]: [effect]`.
     - Add a creature type → `is a Beast in addition to its other types` (or `is also a Beast`).
     - Toggle via a counter → `[cost]: Put a [name] counter on this creature. If it already has one, remove it instead.`

3. **Decide the verdict by re-reading the text you actually emitted, not your intentions.** Read your revised `reminder_text` and both example cards back as if you were a fresh reviewer. "OK" only if they are now genuinely sound, consistent, and rules-complete; "REVISE" if ANY consensus issue still stands in the text you wrote (it will get another round). When in doubt, REVISE. The council will independently re-review your revision, so an over-confident "OK" only wastes a round.

Submit with the `submit_mechanic_synthesis` tool:
- `synthesis`: one or two sentences on what the council agreed on.
- `consensus_issues`: the issues you acted on, each with its `agreement` count.
- `revised_mechanic`: the complete improved mechanic (or the original unchanged if the council found nothing real).
- `verdict`: OK or REVISE, judging the revised mechanic.
- `review_notes`: one or two sentences naming what you changed and why (empty string if you changed nothing).
